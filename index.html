The Three Pillars of The Bond

A novel approach to state synchronization, data fetching, and micro-frontend orchestration within React applications.

Declarative State Linking

Define your shared state once, and let the Bonding Core handle real-time synchronization across the entire component tree, regardless of nesting depth or micro-app boundary.

Cross-Protocol Data Fabric

Unify data access from GraphQL, REST, and WebSockets under one `useBond` hook. The Fabric ensures optimal caching, consistency, and minimal network traffic automatically.

Micro-Frontend Orchestration

Simplify the deployment and runtime communication between independent React applications (Module Federation or similar), ensuring they interact as if they were a single page.

Integration Simplicity: The useBond Hook

Our SDK is engineered for effortless adoption. Replace complex Redux or Context boilerplates with a single, highly performant hook that manages shared state and MFE data fabric. 


import React from 'react';
import { useBond } from '@reactbond/sdk';
import { UserProfileBond } from './bonds/UserBonds'; // Your defined global state schema

// Component 1: A deep-nested header (MFE A)
const HeaderComponent = () => {
  const { user, isLoggedIn } = useBond(UserProfileBond);

  if (!isLoggedIn) return (<button>Login</button>);

  return (
    <div>
      // Real-time state from the Bonding Layer
      Welcome back, {user.firstName}!
    </div>
  );
};

// Component 2: A profile editor (MFE B)
const ProfileEditor = ({ userId }) => {
  const { user, updateBond } = useBond(UserProfileBond);
  
  const handleSave = (newDetails) => {
    // Write operation instantly syncs across all bonded components (MFEs)
    updateBond({ ...user, ...newDetails });
  };

  return (<ProfileForm onSave={handleSave} />);
};
